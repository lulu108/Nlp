# Skill: P4 Experiment Checker

Use this skill to verify whether the P4 project satisfies the Experiment 4 requirements.

## Goal

Check feature completeness, demo readiness, and report readiness.

## Required Checks

- web interface exists
- user interaction is supported
- input text can produce tokenization results
- input text can produce NER results
- input text can produce classification results
- multiple documents can be clustered
- clustering results are visualized
- basic system tests exist
- the project can support screenshots and report writing
- check whether API_SPEC and README are aligned with implementation
- check whether the README demo sequence can run end-to-end in practice

## Review Method

When using this skill:

1. inspect implemented modules
2. compare them against experiment requirements
3. identify missing items
4. suggest concrete fixes
5. summarize demo flow

## Output Expectations

- checklist style summary
- missing feature list
- weak points in demo readiness
- suggestions for screenshots and report materials

## Notes

- Focus on whether the system matches the course experiment requirements
- Do not focus only on code quality
- Demo completeness is important
